- Confirmed **Python 3.14** + **Pillow (PIL) 12.2** are available ‚Äî used to auto-generate placeholder art.

### 2. Created a scalable, professional folder structure
Set up all the folders a full game needs, even though some are empty for now (reserved for future milestones).

Wrote a Python script that produces simple, fully-original PNG art:
- Player character ("Lumi", a friendly teal blob)

- Coin
- Sky background gradient

### 4. Built a reusable Player scene + movement system
A self-contained `Player.tscn` (body + sprite + collision + camera) driven by `Player.gd`, featuring:
- Running with smooth acceleration/friction
- Gravity + terminal fall speed
- Single jump **and** double jump
- Variable jump height (tap = small hop, hold = full jump)
- **Coyote time** (jump shortly after leaving a ledge)
- **Jump buffering** (early jump press is remembered)
- Sprite flipping to face movement direction
- Smooth follow **Camera2D**

### 5. Built the first playable level

- A wide ground floor (collision)
- Three floating platforms to test jumping
- A fixed sky background (in a `CanvasLayer`)
- An instance of the Player

### 6. Configured the project
- `project.godot` with window size, main scene, GL-compatibility renderer, and a custom **Input Map** (A/D, arrow keys, Space/Up to jump).

### 7. Project hygiene

## Core concepts I learned

- **Scene**: a saved tree of nodes (`.tscn`). The Player and the Level are both scenes.
- **Node**: a single building block with one job (sprite, body, camera, collision shape).
- **Scene instancing**: a scene can be dropped inside another scene and reused (Player inside Level).
- **CharacterBody2D**: a physics body designed for player-controlled movement.
- **StaticBody2D**: a body that never moves (ground and platforms).
- **CollisionShape2D**: the invisible shape used for physics collisions.
- **`_physics_process(delta)`**: runs at a fixed physics rate; where movement code belongs.
- **`velocity` + `move_and_slide()`**: set a speed, Godot moves the body and resolves collisions.
- **`@export`**: exposes a variable in the Inspector so values can be tuned without editing code.
- **In 2D, Y increases downward**: "up" is negative, which is why `jump_velocity` is negative.
- **Coyote time / jump buffering / variable jump height**: small forgiveness tricks that make jumping feel good.

## How to run

1. Install Godot 4 (Standard) from godotengine.org ‚Üí drag `Godot.app` to Applications.
2. In the Project Manager, click **Import** and select `project.godot` in this folder ‚Üí **Import & Edit**.
3. Press **F5** to run. Move with A/D or arrows; jump with Space (press again mid-air to double jump).

To regenerate placeholder art at any time:
```bash
python3 scripts/generate_placeholders.py
```

## How to debug

- **Output panel** (bottom): use `print(...)` to inspect values while playing.
- **Debugger panel**: shows red errors with the exact line and a stack trace on crash.
- Common beginner issues:
  - *Null instance error* ‚Üí a node path like `$Sprite2D` doesn't match the scene tree.
  - *Falls through floor* ‚Üí a `CollisionShape2D` is missing or has no shape assigned.
  - *Nothing moves* ‚Üí check **Project ‚Üí Project Settings ‚Üí Input Map**.
